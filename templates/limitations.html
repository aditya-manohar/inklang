Inklang Limitations & Upcoming Features

Inklang is a minimalist language for machine learning, designed for simplicity. Below, we outline the current limitations and the planned features that will address them in future versions to make Inklang a comprehensive DSL for ML and data processing.

Current Limitations

Inklang provides a solid foundation for machine learning workflows, but it is still evolving. Below are the current limitations, what users can do now, and how these constraints limit practical use. Each limitation is accompanied by planned features to overcome it.

Limited Model Types

What You Can Do Now: Train models using NeuralNetwork, RandomForest, SVM, XGBoost, LinearRegression, or KMeans. For example:

acc = train svm1 with iris using SVM for 10 epochs
acc = train lr1 with iris using LinearRegression for 10 epochs
acc = train km1 with iris using KMeans for 10 epochs

Limitation: Only six model types are supported, covering basic classification, regression, and clustering. Advanced models like decision trees, gradient boosting variants (beyond XGBoost), or deep learning architectures (e.g., CNNs, RNNs) are not available. This restricts use for complex tasks like image recognition or NLP.

Upcoming Features:

  • Add support for advanced models like Convolutional Neural Networks (CNNs), Recurrent Neural Networks (RNNs), and Decision Trees in the train statement.
  • Integrate with libraries like Hugging Face Transformers for pre-trained NLP models (e.g., train bert1 with dataset using BERT).
  • Allow custom model definitions via external scripts or configuration files.

No Hyperparameter Tuning

What You Can Do Now: Train models with default settings (e.g., 100 trees for RandomForest, 10 clusters for KMeans). For example: 

acc = train rf1 with iris using RandomForest for 10 epochs

Limitation: No ability to specify hyperparameters (e.g., learning rate, number of trees, kernel type for SVM). This limits model optimization and performance tuning, critical for real-world ML tasks.

Upcoming Features:

  • Add hyperparameter clause to train statement (e.g., acc = train rf1 with iris using RandomForest with n_estimators 200 learning_rate 0.01 for 10 epochs).
  • Support grid search or random search for hyperparameter tuning (e.g., grid_search rf1 with iris using RandomForest).

Basic Preprocessing Options

What You Can Do Now: Apply normalization, standardization, sampling, dropping missing values, or one-hot encoding. For example:

preprocess iris with dropna
preprocess iris with normalize to 1.0

Limitation: Limited to five operations. No support for advanced preprocessing like imputation (e.g., mean/median fill), PCA, or feature selection, which are essential for handling noisy or high-dimensional data.

Upcoming Features:

  • Add preprocessing operations like impute_mean, impute_median, pca to 10, and select_kbest to 5.
  • Support custom preprocessing pipelines (e.g., pipeline iris with normalize then pca).

No Data Splitting or Cross-Validation

What You Can Do Now: Train models with an internal 80/20 train-test split, but you cannot control the split or perform cross-validation. For example:

acc = train rf1 with iris using RandomForest for 10 epochs

Limitation: Fixed 80/20 split limits flexibility. No cross-validation support means less robust model evaluation, especially for small datasets.

Upcoming Features:

  • Add split statement (e.g., split iris into train test with ratio 0.8).
  • Add cross_validate statement (e.g., cv_acc = cross_validate rf1 with iris folds 5).

Limited Evaluation Metrics

What You Can Do Now: Use acc variable for accuracy (classification), R² (regression), or silhouette score (clustering). For example: 

acc = train svm1 with iris using SVM for 10 epochs
if acc > 0.9:
    print "Good accuracy!"

Limitation: Only one metric per model type is available. No support for precision, recall, F1-score, or confusion matrices, which are critical for imbalanced datasets or detailed analysis.

Upcoming Features:

  • Add evaluate statement returning a dictionary of metrics (e.g., metrics = evaluate rf1 with iris, accessing metrics.precision).
  • Support confusion matrix output and other metrics like MSE, ROC-AUC.

No Visualization Support

What You Can Do Now: Output text results via print (e.g., predictions, metrics). For example:

predict rf1 with iris

Limitation: No way to visualize data distributions, feature correlations, or model performance (e.g., confusion matrices, ROC curves), limiting data exploration and interpretability.

Upcoming Features:

  • Add plot statement for data and metrics (e.g., plot iris column "feature1", plot confusion rf1 with iris).
  • Generate and display images in the playground using Matplotlib or Seaborn.

Limited Variable and Expression Support

What You Can Do Now: Use acc variable for model metrics in if statements. For example: 

if acc > 0.9:
    print "High accuracy!"

Limitation: Only acc is supported; no custom variables or mathematical expressions (e.g., acc * 100), limiting complex logic or parameter reuse.

Upcoming Features:

  • Add var statement for custom variables (e.g., var ratio = 0.8).
  • Support basic expressions (e.g., var score = acc * 100).

No Model Deployment

What You Can Do Now: Save models to pickle files and load them for predictions. For example:

savemodel rf1 to "rf1.pkl"
loadmodel rf1 from "rf1.pkl"

Limitation: No support for deploying models as APIs or integrating with production environments (e.g., Flask, ONNX, or cloud services), limiting real-world use.

Upcoming Features:

  • Add deploy statement (e.g., deploy rf1 to "api_endpoint" format "onnx").
  • Generate serving code (e.g., Flask/FastAPI) or export to ONNX/TensorFlow Serving.

No Support for Large or Streaming Data

What You Can Do Now: Load entire datasets (CSV, JSON, or built-in) into memory. For example:

dataset mnist from "mnist"

Limitation: Loading full datasets limits scalability for large datasets (e.g., >1GB) or streaming data, causing memory issues.

Upcoming Features:

  • Add batch_load statement (e.g., batch_load dataset from "large.csv" size 1000).
  • Support incremental training for streaming data.

Limited Integration with External Tools

What You Can Do Now: Work with local CSV/JSON files or built-in datasets (mnist, fashion_mnist, cifar10). For example:

dataset iris from "iris.csv"

Limitation: No integration with external ML platforms (e.g., Hugging Face, cloud storage like S3), limiting access to pre-trained models or remote data.

Upcoming Features:

  • Add import statement for external models (e.g., import model from "huggingface:bert-base").
  • Support cloud storage access (e.g., dataset data from "s3://bucket/data.csv").

Future Vision

Inklang aims to be a comprehensive DSL for machine learning and data processing, balancing simplicity with power. Future versions will address these limitations to support:

  • Advanced ML workflows, including deep learning, NLP, and computer vision.
  • Production-ready deployment with APIs and cloud integration.
  • Scalable data handling for big data and real-time applications.
  • Rich visualization and evaluation tools for better interpretability.
  • Community-driven extensions via external model imports and custom scripts.

We are committed to evolving Inklang based on user feedback. Join our GitHub or Discord to contribute ideas or report issues.
