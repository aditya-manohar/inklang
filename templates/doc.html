Inklang Documentation

Learn the syntax of Inklang, a minimalist language for training and deploying machine learning models with elegant syntax.

Syntax Overview

Inklang is designed for simplicity, focusing on machine learning workflows. It supports statements for loading datasets, preprocessing data, training models, making predictions, and controlling flow with conditionals and loops. Below is a detailed guide to each statement, including syntax, usage, and examples.

General Rules

  • Tokens: Identifiers (NAME: e.g., mnist, nn1), quoted strings (STRING: e.g., "iris.csv"), numbers (NUMBER: e.g., 10, 0.95), and comparison operators (CMP_OP: >, <, ==, >=, <=, !=).
  • Comments: Single-line comments start with # (e.g., # Example).
  • Indentation: Not required for parsing (whitespace is ignored), but recommended (4 spaces or 1 tab) for readability in if and repeat blocks.
  • Block Delimiters: Use : to start blocks in if and repeat statements.

Statements

Print Statement

Outputs a message to the console or website output.

Syntax:

print STRING

Example:

print "Hello, Inklang!"

Output: [inklang] Hello, Inklang!

Dataset Statement

Loads a dataset into memory from a file (CSV or JSON) or a built-in dataset (e.g., mnist, fashion_mnist, cifar10). The dataset must have a label column as the target variable.

Syntax:

dataset NAME from STRING

Example:

dataset iris from "iris.csv"
dataset mnist from "mnist"

Notes: NAME is the dataset identifier; STRING is a file path (e.g., "iris.csv") or built-in dataset name (e.g., "mnist").

Preprocess Statement

Applies preprocessing operations to a dataset, such as normalization, standardization, sampling, dropping missing values, or one-hot encoding categorical features.

Syntax:

preprocess NAME with PREPROCESS_OP ["to" NUMBER]

Supported Operations:

  • normalize: Scales features to a range (default [0, 1]; specify max with to NUMBER).
  • standardize: Scales features to zero mean and unit variance (no to NUMBER).
  • sample: Randomly samples a fraction of rows (requires to NUMBER between 0 and 1).
  • dropna: Removes rows with missing values (no to NUMBER).
  • onehot: One-hot encodes categorical columns (no to NUMBER).

Example:

dataset iris from "iris.csv"
preprocess iris with dropna
preprocess iris with sample to 0.5
preprocess iris with normalize to 1.0
preprocess iris with onehot
preprocess iris with standardize

Notes: Operations are applied in sequence; to NUMBER is required for sample and optional for normalize. Ensure the dataset has a label column if used for training.

Train Statement

Trains a model on a dataset and stores its accuracy in the acc variable. Supports NeuralNetwork (default) and RandomForest.

Syntax:

acc = train NAME with NAME ["using" NAME] for NUMBER epochs

Example:

acc = train nn1 with mnist using NeuralNetwork for 10 epochs
acc = train rf1 with iris using RandomForest for 100 epochs

Notes: using NAME is optional (defaults to NeuralNetwork); NUMBER must be an integer for epochs. Dataset must have a label column.

Predict Statement

Makes predictions using a trained model on a dataset and outputs the results.

Syntax:

predict NAME with NAME

Example:

predict rf1 with iris

Notes: Outputs up to 10 predictions followed by the total count. Dataset may or may not have a label column.

Save Model Statement

Saves a trained model to a file in pickle format (.pkl).

Syntax:

savemodel NAME to STRING

Example:

savemodel rf1 to "rf1.pkl"

Notes: The file is saved in the Uploads directory and automatically downloaded in the playground.

Load Model Statement

Loads a model from a pickle file (.pkl).

Syntax:

loadmodel NAME from STRING

Example:

loadmodel rf1 from "rf1.pkl"

Notes: The file must exist in the Uploads directory.

If Statement

Executes a block of statements if the acc variable satisfies a condition.

Syntax:

if acc CMP_OP NUMBER :
    statement+

Example:

if acc > 0.95:
    print "Excellent accuracy!"
    savemodel rf1 to "rf1.pkl"

Notes: CMP_OP is >, <, ==, >=, <=, or !=. Indentation is recommended but not required.

Repeat Statement

Repeats a block of statements a fixed number of times.

Syntax:

repeat NUMBER times :
    statement+

Example:

repeat 3 times:
    print "Training iteration"
    acc = train nn1 with mnist for 5 epochs

Notes: NUMBER must be an integer. Indentation is recommended but not required.

Quick Reference

# Print a message
print "Hello, Inklang!"

# Load dataset
dataset NAME from STRING

# Preprocess dataset
preprocess NAME with (normalize | standardize | sample | dropna | onehot) ["to" NUMBER]

# Train model
acc = train NAME with NAME [using (NeuralNetwork | RandomForest)] for NUMBER epochs

# Make predictions
predict NAME with NAME

# Save/load models
savemodel NAME to STRING
loadmodel NAME from STRING

# Conditional logic
if acc CMP_OP NUMBER :
    statement+

# Repeat loop
repeat NUMBER times :
    statement+

# Comments
# This is a comment

Notes: Indentation is optional but recommended (4 spaces or 1 tab) for readability in if and repeat blocks. Datasets must have a label column for training. The to NUMBER clause is required for sample, optional for normalize, and not allowed for standardize, dropna, or onehot.

Supported Datasets and Models

Built-in Datasets

  • mnist: Handwritten digits (28x28 grayscale images, 10 classes).
  • fashion_mnist: Clothing items (28x28 grayscale images, 10 classes).
  • cifar10: Small images (32x32 color images, 10 classes).

Notes: Use the dataset name as the STRING in dataset NAME from STRING (e.g., dataset mnist from "mnist").

File-based Datasets

Load datasets from CSV or JSON files. The file must have a label column for training. Files should be uploaded to the Uploads directory.

Example:

dataset iris from "iris.csv"

Supported Models

  • NeuralNetwork: A feedforward neural network with one hidden layer (128 units, ReLU activation) and dropout (0.2).
  • RandomForest: A random forest classifier with 100 trees.

Notes: Specify the model type in the using NAME clause of the train statement. If omitted, defaults to NeuralNetwork.

Example Workflow

A complete example demonstrating preprocessing, training, and conditional logic:

# Load and preprocess dataset
dataset iris from "iris.csv"
preprocess iris with dropna
preprocess iris with sample to 0.5
preprocess iris with normalize to 1.0

# Train model
acc = train rf1 with iris using RandomForest for 10 epochs

# Check accuracy
if acc > 0.9:
    print "Great accuracy!"
    savemodel rf1 to "rf1.pkl"

# Make predictions
predict rf1 with iris

Notes: This workflow loads the Iris dataset, removes missing values, samples 50% of the rows, normalizes features, trains a RandomForest model, saves it if accuracy exceeds 0.9, and makes predictions.
